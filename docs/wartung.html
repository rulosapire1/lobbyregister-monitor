Wartungsdokumentation

Lobbyregister-Monitor · Interne Dokumentation

Wichtig: Echte Zugangsdaten sind ausschließlich als GitHub Secrets hinterlegt.

1. Systemübersicht

Der Lobbyregister-Monitor ruft täglich inkrementell Stellungnahmen ab:

  • Nur neue Registereinträge werden von der API geladen (bereits bekannte werden übersprungen)
  • KI-Prüfung (Gemini 3.1 Flash Lite) mit lokalem Cache – bereits geprüfte Einträge werden nicht erneut gesendet
  • Budgetlimit: max. 100 KI-Calls pro Durchlauf
  • Übersichtsseite auf GitHub Pages mit Pagination

Zusätzlich jeden Montag: Wochenmail und Selbsttest.

Ablauf: täglich um 6:00 Uhr MESZ (4:00 UTC).

4. Ablauf eines täglichen Durchlaufs

  1. Cache laden – data.json, gemini_cache.json, run_history.json aus GitHub Actions Cache
  2. fetch_and_build.py – Registernummern laden, nur neue einzeln abrufen, mit vorherigen Daten mergen
  3. gemini_enrich.py – Neue Einträge per KI prüfen (max. 100/Run), gecachte überspringen, HTML neu bauen
  4. save_run_log.py – Durchlauf-Statistiken an run_history.json anhängen
  5. Deploy – GitHub Pages veröffentlichen
  6. Montags: send_email.py (Wochenmail) + health_check.py (Selbsttest)

5. Fehlersuche

Seite nicht aktualisiert

  1. GitHub Actions öffnen
  2. Fehlgeschlagenen Job anklicken und Logs lesen
  3. Häufig: API-Key abgelaufen, GitHub Pages nicht aktiviert

Gemini fehlgeschlagen

Erkennbar an „KI-Prüfung ausstehend“-Badge auf Karten oder gelbem Warnbanner.

  • Quota: 500 req/Tag – Caching und Budgetlimit (100/Run) reduzieren den Verbrauch. Ausstehende Prüfungen werden automatisch nachgeholt.
  • Key abgelaufen: Neuen Key auf aistudio.google.com erstellen
  • Modell geändert: GEMINI_MODEL in gemini_enrich.py und health_check.py anpassen

Manueller Start

Repository → Actions → „Lobbyregister Monitor“ → Run workflow.

6. System neu aufsetzen

  1. GitHub-Account: github.com/signup
  2. Repository anlegen: lobbyregister-monitor, Public
  3. GitHub Pages aktivieren: Settings → Pages → Source: GitHub Actions
  4. Secrets eintragen (siehe Abschnitt 2)
  5. Dateien aus dem alten Repository übertragen
  6. Resend: Account auf resend.com, Domain verifizieren
  7. Gemini: Key auf aistudio.google.com
  8. URLs in update.yml, fetch_and_build.py, template.html anpassen
